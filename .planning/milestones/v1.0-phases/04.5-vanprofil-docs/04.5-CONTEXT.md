# Phase 4.5: Vänprofil-navigation och dokumentationsfix - Context

**Gathered:** 2026-03-03
**Status:** Ready for planning

<domain>
## Phase Boundary

Användare kan navigera till en väns profil från UI:t + dokumentationsgap fixade (SUMMARY-frontmatter och REQUIREMENTS-traceability). ProfileView existerar redan för egen profil — denna fas gör den nåbar för vänner och förenar upplevelsen.

</domain>

<decisions>
## Implementation Decisions

### Navigationsflöde
- Profilen nås ENBART via WeatherDetailSheet — inte direkt från FriendListView
- Tap-target: hela headern (profilbild + namn + stad) i WeatherDetailSheet
- Presentation: ny sheet ovanpå WeatherDetailSheet (stackade sheets)
- Nuvarande tap-beteende i FriendListView (tap → WeatherDetailSheet) behålls oförändrat

### Profilinnehåll
- Grundläggande info: profilbild, namn, stad — inga extra detaljer
- Samma vy för alla vänner oavsett om de har appkonto eller inte (Friend-data, inte AppUser-data)
- Enhetlig profilupplevelse: egen profil och vänprofil ska kännas som samma typ av vy
- Egen profil visar "Redigera profil"-knapp, vänprofil visar ingen åtgärdsknapp

### Åtgärder på profilen
- Ren visningsvy — inga åtgärdsknappar på vänprofil
- "Redigera profil"-knapp visas bara för egen profil (befintligt beteende)

### Dokumentationsfix
- AUTH-01, AUTH-02, AUTH-03 ska finnas i 01-02-SUMMARY.md frontmatter (requirements-fält)
- REQUIREMENTS.md traceability-tabell ska visa korrekt status för alla lösta krav

### Claude's Discretion
- Visuell indikering att headern är tappbar (chevron, ingen markering, eller annan approach)
- Implementationsapproach: ny FriendProfileView vs utökad ProfileView
- Favorit-stjärna på profilvyn (visa eller ej)
- Stäng-knapp vs bara dra-ner (matcha appens övriga sheets)

</decisions>

<specifics>
## Specific Ideas

- Profilvyn ska kännas som samma vy oavsett om det är egen profil eller en väns — inte två separata upplevelser
- Nuvarande WeatherDetailSheet-header har redan profilbild (80px), namn och stad — tap-target är naturligt

</specifics>

<code_context>
## Existing Code Insights

### Reusable Assets
- `ProfileView` (Features/Profile/ProfileView.swift): Befintlig profilvy som tar `uid: String`, laddar `AppUser` via `UserService`. Visar profilbild, namn, stad, redigera-knapp. Använder `.presentationDetents([.medium])` och `.presentationDragIndicator(.visible)`.
- `ProfileViewModel` (Features/Profile/ProfileViewModel.swift): Laddar AppUser-data via UserService
- `WeatherDetailSheet` header-sektion: Redan profilbild (80px Circle), displayName, city — kan göras tappbar
- `initialsCircle()`: Finns i både ProfileView och WeatherDetailSheet — kan extraheras

### Established Patterns
- Sheets med `.presentationDetents([.medium])` och `.presentationDragIndicator(.visible)` — konsekvent i hela appen
- `@Environment`-injektion av services (AuthManager, UserService, FriendService)
- `Friend`-modellen har `authUid: String?` — nil för vänner utan appkonto

### Integration Points
- WeatherDetailSheet → lägg till tap gesture på headerSection → öppna profil-sheet
- Friend-modellen innehåller alla fält som behövs (displayName, photoURL, city)
- FriendListView → ingen ändring behövs (tap-beteende behålls)

</code_context>
