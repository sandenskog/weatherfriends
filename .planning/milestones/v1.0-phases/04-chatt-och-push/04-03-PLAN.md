**chatPushTrigger.ts** — Firestore trigger vid nytt meddelande:

```typescript
import { onDocumentCreated } from "firebase-functions/v2/firestore";
import { getFirestore } from "firebase-admin/firestore";
import { getMessaging } from "firebase-admin/messaging";
import { initializeApp, getApps } from "firebase-admin/app";

if (getApps().length === 0) initializeApp();
```

Exportera `onNewMessage`:
- Trigger: `onDocumentCreated({ document: "conversations/{conversationId}/messages/{messageId}", region: "europe-west1" })`
- Steg:
  1. Hamta message data fran event.data.
  2. Hamta conversationId fran event.params.
  3. Hamta konversationsdokumentet for att fa participants-array.
  4. Filtrera bort senderId fran participants — kvarstaende ar mottagare.
  5. **Blockerings-kontroll:** For varje mottagare, kontrollera om avsandaren finns i mottagarens blockedUsers-subkollektion. Om blockerad — skippa den mottagaren.
  6. Hamta FCM-tokens for alla ej-blockerade mottagare fran users/{uid}.fcmToken.
  7. Filtrera bort null/tomma tokens.
  8. Hamta avsandarens displayName fran users/{senderId}.
  9. Bestam body-text: om type == "weatherSticker" -> "Skickade en vader-sticker" (med molnikon), annars message.text.
  10. Skicka via `getMessaging().sendEachForMulticast()`:
      ```typescript
      {
        tokens,
        notification: {
          title: senderName,
          body: bodyText,
        },
        data: {
          type: "chat",
          conversationId,
        },
        apns: {
          payload: {
            aps: { sound: "default" },
          },
        },
      }
      ```
  11. Hantera TokenNotRegistered-fel: om nagon token ar ogiltig, ta bort den fran users-dokumentet (setData med fcmToken: FieldValue.delete()).

**weatherAlertScheduler.ts** — Schemalagd extremvader-check:

PRAGMATISK APPROACH (fran RESEARCH.md Open Question 1): Istallet for att anropa WeatherKit REST API fran Cloud Functions (krav JWT med .p8-nyckel — komplex setup), anvand en ENKLARE strategi:

Alternativ A (rekommenderas for v1):
- iOS-klienten kontrollerar weatherAlerts vid app-start/background refresh och sparar `hasActiveAlert: true/false` + `alertSummary: "Storm"` per van i Firestore (`users/{uid}/friends/{friendId}`).
- Cloud Function triggas av Firestore-andring (`onDocumentUpdated`) nar hasActiveAlert andras fran false till true.
- Skickar push-notis till vanens agare.

Alternativ B (full server-side, komplex):
- Cloud Function anropar WeatherKit REST API med JWT (krav Apple .p8-nyckel som Firebase Secret).
- Itererar over alla anvandare och deras vanner.

**Implementera Alternativ A (enklare):**

```typescript
import { onSchedule } from "firebase-functions/v2/scheduler";

```

Exportera `checkExtremeWeather`:
- Schema: `onSchedule({ schedule: "every 60 minutes", region: "europe-west1" })`
- Logik:
  1. Hamta alla users med fcmToken != null.
  2. For varje user: hamta friends-subkollektion.
  3. For varje van med koordinater: gora en enkel temperaturkontroll (extremt kall < -20, extremt varmt > 40). OBS: Detta ar en FORELOPIG implementation — full WeatherKit alerts krav iOS-klienten som satter hasActiveAlert.
  4. Rate-limit: kolla `lastAlertSentAt` pa friend-dokumentet. Om < 24h sedan, skippa.
  5. Om alert ska skickas: skicka FCM till user med personlig ton: "Extremt vader hos [vannens namn] i [stad]! Hor av dig!"
  6. Uppdatera `lastAlertSentAt` pa friend-dokumentet.

Cloud Functions for push-notiser ar klara. For att push faktiskt ska na iOS-enheter behovs en APNs Authentication Key uppladdad till Firebase Console.

**Steg 1: Skapa APNs Key i Apple Developer Portal**
1. Ga till https://developer.apple.com/account/resources/authkeys/list
2. Klicka "+" (Create a Key)
3. Ge nyckeln ett namn (t.ex. "HotAndColdFriends Push")
4. Bocka i "Apple Push Notifications service (APNs)"
5. Klicka "Continue" och sedan "Register"
6. Ladda ner .p8-filen (SPARA DEN — den kan bara laddas ner EN gang)
7. Notera Key ID (visas pa sidan)

**Steg 2: Ladda upp till Firebase Console**
1. Ga till https://console.firebase.google.com
2. Valj ditt projekt (HotAndColdFriends)
3. Klicka kugghjulet (Settings) -> "Project settings"
4. Valj fliken "Cloud Messaging"
5. Under "Apple app configuration" -> "APNs Authentication Key"
6. Klicka "Upload" (eller "Add" om det ar forsta gangen)
7. Valj .p8-filen du laddade ner
8. Fyll i Key ID (fran steg 1.7)
9. Fyll i Team ID: A473BQKT8M
10. Klicka "Upload"
