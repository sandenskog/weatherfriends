## Summary

Fasen är en ren iOS/SwiftUI-refaktorering utan nya externa beroenden. Problemet är exakt lokaliserat: `ContactImportOnboardingWrapper` (rad 351 i `OnboardingFavoritesView.swift`) är en förenklad kopia av `ContactImportView` som saknar hela guessLocations()-ImportReviewView-flödet. Den lägger direkt till valda kontakter som `PendingFriend` med `cityLatitude: nil, cityLongitude: nil`, vilket gör att `AppWeatherService` inte kan returnera väderdata för dessa vänner.

Det finns ett parallellt gap i `ImportReviewView.buildReviewItems()` (rad 251-265): metoden bygger `ReviewItem`-objekt men sätter alltid `latitude: nil, longitude: nil` — geocoding sker aldrig automatiskt. Användaren måste idag manuellt söka stad för att få koordinater, även om AI-gissningen returnerade en korrekt stad. Detta påverkar BÅDA flödena (onboarding och standard ContactImportView) och ska fixas i samma fas.

Lösningen har två delar: (1) Utöka `ContactImportOnboardingWrapper` med `guessLocations()` + `isGuessing`-state + sheet till `ImportReviewView` i PendingFriend-läge, och (2) Lägg till auto-geocoding i `ImportReviewView.buildReviewItems()` via `CLGeocoder.geocodeAddressString()` för kontakter med stad men utan koordinater.

**Primary recommendation:** Refaktorera `ImportReviewView` med ett `mode`-enum (`onboarding(callback: ([PendingFriend]) -> Void)` vs `standard(uid:friendService:)`), lägg till async geocoding i `buildReviewItems()`, och utöka `ContactImportOnboardingWrapper` till att spegla `ContactImportView`s importSelected()-flöde.

### Pattern 1: ImportReviewView mode-enum

**What:** Lägg till ett `mode`-enum som avgör vad som händer vid "Spara"
**When to use:** När samma vy ska ha olika sparbeteenden beroende på kontext

```swift
// Rekommenderat mönster:
enum ImportReviewMode {
    case standard(uid: String, friendService: FriendService)
    case onboarding(onPendingFriendsReady: ([PendingFriend]) -> Void)
}

struct ImportReviewView: View {
    let mode: ImportReviewMode
    let contacts: [ImportableContact]
    let locationGuesses: [LocationGuess]
    let contactImportService: ContactImportService
    let onCompleted: () -> Void
    // uid och friendService tas bort som direkta properties — hämtas via mode
}
```

### Pattern 2: Async geocoding i buildReviewItems()

**What:** Geocoda AI-gissningarnas stad-strängar till koordinater automatiskt
**When to use:** Direkt i .task { buildReviewItems() } — innan review-vyn visas

```swift
// CLGeocoder för batch-geocoding av stadsnamn:
private func buildReviewItems() async {
    var items: [ReviewItem] = contacts.map { contact in
        let guess = locationGuesses.first { $0.identifier == contact.id }
        return ReviewItem(
            id: contact.id,
            contact: contact,
            city: guess?.city ?? "",
            country: guess?.country ?? "",
            latitude: nil,
            longitude: nil,
            confidence: guess?.confidence ?? "unknown",
            reason: guess?.reason ?? "Ingen data tillgänglig",
            isIncluded: true
        )
    }

    // Auto-geocoda alla items med stad men utan koordinater
    let geocoder = CLGeocoder()
    for index in items.indices {
        let item = items[index]
        guard !item.city.isEmpty, item.latitude == nil else { continue }

        let addressString = item.country.isEmpty
            ? item.city
            : "\(item.city), \(item.country)"

        if let placemark = try? await geocoder.geocodeAddressString(addressString).first,
           let location = placemark.location {
            items[index].latitude = location.coordinate.latitude
            items[index].longitude = location.coordinate.longitude
        }
        // Misslyckas: koordinater förblir nil → röd prick → manuell sökning
    }

    reviewItems = items
}
```

**Viktigt:** `CLGeocoder.geocodeAddressString()` är inte `nonisolated` — anropa från `@MainActor` (t.ex. i `.task { }` på vyn). Begränsa anrop till ett åt gången (CLGeocoder tillåter inte parallella anrop från samma instans).

### Pattern 3: ContactImportOnboardingWrapper med guessLocations()

**What:** Spegla ContactImportView.importSelected() — lägg till isGuessing-state och sheet till ImportReviewView
**When to use:** När användaren trycker "Importera (N)" i onboarding-wrappern

```swift
// Lägg till i ContactImportOnboardingWrapper:
@State private var isGuessing = false
@State private var showReview = false
@State private var selectedContacts: [ImportableContact] = []
@State private var locationGuesses: [LocationGuess] = []

// Toolbar-knappen (speglar ContactImportView exakt):
Button {
    Task { await importSelected() }
} label: {
    if isGuessing {
        HStack(spacing: 4) {
            ProgressView().controlSize(.small)
            Text("Analyserar...")
        }
    } else {
        Text("Importera (\(selectedIds.count))")
            .fontWeight(.semibold)
    }
}
.disabled(selectedIds.isEmpty || isGuessing)

// importSelected()-metoden:
private func importSelected() async {
    isGuessing = true
    defer { isGuessing = false }
    selectedContacts = contacts.filter { selectedIds.contains($0.id) }
    do {
        locationGuesses = try await contactImportService.guessLocations(for: selectedContacts)
    } catch {
        locationGuesses = []  // Visa review ändå, utan gissningar
    }
    showReview = true
}

// Sheet med ImportReviewView i onboarding-läge:
.sheet(isPresented: $showReview) {
    ImportReviewView(
        mode: .onboarding { newPending in
            pendingFriends.append(contentsOf: newPending)
        },
        contacts: selectedContacts,
        locationGuesses: locationGuesses,
        contactImportService: contactImportService
    ) {
        dismiss()
    }
}
```

### Pattern 4: saveAll() med mode-switch

**What:** ImportReviewView.saveAll() branschar på mode för att antingen spara till Firestore eller returnera PendingFriends via callback

```swift
private func saveAll() async {
    isSaving = true
    defer { isSaving = false }

    let included = reviewItems.filter { $0.isIncluded }

    switch mode {
    case .standard(let uid, let friendService):
        // Befintligt Firestore-flöde
        let reviewedContacts = included.map { item in
            ReviewedContact(
                contact: item.contact,
                city: item.city,
                country: item.country,
                latitude: item.latitude,
                longitude: item.longitude,
                confidence: item.confidence
            )
        }
        do {
            try await contactImportService.saveReviewedContacts(
                uid: uid,
                reviewedContacts: reviewedContacts,
                friendService: friendService
            )
            onCompleted()
            dismiss()
        } catch {
            errorMessage = "Kunde inte spara kontakter: \(error.localizedDescription)"
        }

    case .onboarding(let onPendingFriendsReady):
        // Nytt PendingFriend-flöde
        let pendingFriends = included.map { item -> PendingFriend in
            let cityDisplay = item.city.isEmpty
                ? "Okänd plats"
                : (item.country.isEmpty ? item.city : "\(item.city), \(item.country)")
            return PendingFriend(
                displayName: item.contact.fullName,
                city: cityDisplay,
                cityLatitude: item.latitude,
                cityLongitude: item.longitude
            )
        }
        onPendingFriendsReady(pendingFriends)
        onCompleted()
        dismiss()
    }
}
```

### Anti-Patterns to Avoid

- **Parallell CLGeocoder:** CLGeocoder tillåter inte flera samtida anrop från samma instans (`Error Domain=kCLErrorDomain Code=3 "geocodeAddressString"` om du inte väntar på föregående). Geocoda sekventiellt med `for`-loop, inte `async let` eller `withTaskGroup`.
- **Geocoda i viewModel:** Håll geocoding i `ImportReviewView` (vyn äger sin lokala state) — flytta inte ut till en ViewModel som inte finns i detta projekt.
- **Bryta ContactImportView:** Mode-enum måste vara bakåtkompatibelt — `ContactImportView` ska inte behöva ändras förutom att byta `uid: uid, friendService: friendService` till `mode: .standard(uid: uid, friendService: friendService)`.

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Geocoding av stad-strängar | Custom HTTP-anrop till Google Maps | CLGeocoder.geocodeAddressString() | Apple-inbyggt, gratis, fungerar offline, kräver inget API-nyckel |

**Key insight:** Hela tekniken finns redan — gapet är att onboarding-wrappern aldrig anropar guessLocations() och att buildReviewItems() aldrig geocodar.

## Common Pitfalls

### Pitfall 1: CLGeocoder sekventialitetskrav
**What goes wrong:** `CLGeocoder` kraschar eller returnerar `kCLErrorDomain Code=3` om du startar ett nytt anrop innan föregående är klart.
**Why it happens:** CLGeocoder-instansen kan bara hantera ett aktivt anrop åt gången.
**How to avoid:** Skapa en ny `CLGeocoder`-instans per geocodning, ELLER använd en sekventiell `for`-loop (inte `async let`/`withTaskGroup`).
**Warning signs:** Intermittenta nil-koordinater i buildReviewItems trots korrekt stad-sträng.

```swift
// Säkert mönster — ny instans per anrop:
for index in items.indices {
    let geocoder = CLGeocoder()  // Ny instans varje gång
    if let placemark = try? await geocoder.geocodeAddressString(addressString).first {
        items[index].latitude = placemark.location?.coordinate.latitude
        items[index].longitude = placemark.location?.coordinate.longitude
    }
}
```

### Pitfall 2: buildReviewItems() är synkron — .task{} krävs
**What goes wrong:** `buildReviewItems()` kallas i `.task { buildReviewItems() }` — om den görs `async` utan att ändra call-site fungerar inte Swift-kompilatorn.
**Why it happens:** `.task { }` förväntar sig `async` closure — men den nuvarande signaturen är `private func buildReviewItems()` (synkron).
**How to avoid:** Ändra signaturen till `private func buildReviewItems() async` och uppdatera call-site till `.task { await buildReviewItems() }`.

### Pitfall 3: PendingFriend-struct är definierad i OnboardingFavoritesView.swift
**What goes wrong:** Om ImportReviewView.swift importerar PendingFriend direkt kompilerar det bara om filen är synlig i module-scopen.
**Why it happens:** `PendingFriend` är en top-level struct i `OnboardingFavoritesView.swift` (inte `private`) — den är synlig i hela appen-modulen. Ingen risk för kompileringsfel.
**How to avoid:** Inga åtgärder krävs — men dokumentera detta om teamet växer.

### Pitfall 4: ContactImportView-anropet till ImportReviewView måste uppdateras
**What goes wrong:** `ContactImportView` anropar `ImportReviewView(uid: uid, contacts: ..., friendService: ...)` — när mode-enum introduceras bryts detta anrop.
**Why it happens:** Signatur-ändring i ImportReviewView.
**How to avoid:** Uppdatera ContactImportView samtidigt som ImportReviewView refaktoreras — det är ett litet ändring (byt parametrarna mot `mode: .standard(uid: uid, friendService: friendService)`).

### Pitfall 5: isGuessing-state i ContactImportOnboardingWrapper stänger av toolbar-knappen
**What goes wrong:** Om `isGuessing = true` men Cloud Function hänger (nätverksfel) är knappen otillgänglig tills timeout.
**Why it happens:** `defer { isGuessing = false }` sätter tillbaka state när Task slutar — oavsett om det är lyckad eller misslyckad.
**How to avoid:** `defer { isGuessing = false }` hanterar detta korrekt — samma mönster som ContactImportView. Inget extra krävs.

## Open Questions

1. **Geocoding-rate limits (MapKit)**
   - What we know: Apple begränsar CLGeocoder till rimliga anrop; exakt gräns är odokumenterad
   - What's unclear: Kommer ett batch på 50 kontakter att throttlas?
   - Recommendation: Geocoda sekventiellt (for-loop) med en ny CLGeocoder-instans per anrop. Lägg en `try? await Task.sleep(nanoseconds: 100_000_000)` (100ms) mellan anrop om problem uppstår. Med max 50 kontakter är detta 5 sekunder — acceptabelt för en engångsoperation vid onboarding.

2. **Visuell laddningsindikator under geocoding i buildReviewItems()**
   - What we know: Geocoding kan ta 2-5 sekunder för 50 kontakter
   - What's unclear: Ska review-vyn visa en spinner medan geocoding sker (innan listan visas)?
   - Recommendation (Claude's discretion): Visa en `ProgressView("Löser upp platser...")` i review-vyn medan `buildReviewItems()` körs, ersätt med List när klart. Samma mönster som ContactImportOnboardingWrapper visar "Hämtar kontakter...".
