**INT-03/FLOW-01 — Deep link race condition i FriendsTabView.swift:**

Ersätt den befintliga `.onChange(of: openWeatherAlertFriendId)` (rad 153-160) med två `onChange`-observers:

1. `.onChange(of: openWeatherAlertFriendId)` — triggar navigation BARA om `viewModel.isLoading == false`. Om loading pågår, gör ingenting — id:t bevaras i `@Binding` och `.onChange(of: viewModel.isLoading)` tar vid.

2. `.onChange(of: viewModel.isLoading)` — ny observer som triggar navigation när loading slutar OCH ett väntande `openWeatherAlertFriendId` finns. Nollställer `openWeatherAlertFriendId` efter navigation (eller om ingen match hittades).

Exakt implementation:
```swift
.onChange(of: openWeatherAlertFriendId) { _, friendId in
    guard let friendId, !viewModel.isLoading else { return }
    let all = viewModel.favorites + viewModel.others
    selectedFriendWeather = all.first(where: { $0.friend.id == friendId })
    openWeatherAlertFriendId = nil
}
.onChange(of: viewModel.isLoading) { _, isLoading in
    guard !isLoading, let friendId = openWeatherAlertFriendId else { return }
    let all = viewModel.favorites + viewModel.others
    selectedFriendWeather = all.first(where: { $0.friend.id == friendId })
    openWeatherAlertFriendId = nil
}
```

VIKTIGT: Ingen extra loading state eller visuell ändring — appen väntar tyst tills data finns. Gäller både push-notis deep links och widget deep links (samma onOpenURL-handler).
