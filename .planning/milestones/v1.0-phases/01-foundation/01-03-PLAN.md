<objective>
Implementera onboarding-wizard (namn → profilbild → stad/land), profilvisning (halvmodal sheet) och profilredigering. Inkluderar LocationService med MKLocalSearchCompleter för stad-autocomplete och CLLocationManager för GPS-förfyllning.

Purpose: Utan komplett profil (namn + plats) har appen ingen data att visa väder för. Onboarding säkerställer att varje ny användare har en fullständig profil. Profilvisning (PROF-03) är nödvändigt för att fas 2+ ska kunna visa vänners information.
Output: Komplett onboarding-flöde, stad-autocomplete, profil-CRUD, och profilvisning.
</objective>

    **LocationService.swift** — @Observable class som conformar till MKLocalSearchCompleterDelegate:
    - Egenskap: `suggestions: [MKLocalSearchCompletion] = []`
    - Egenskap: `queryFragment: String = ""` med didSet som sätter completer.queryFragment
    - Private: `MKLocalSearchCompleter` med `resultTypes = .address`
    - Delegate: `completerDidUpdateResults` → filtrera resultat till städer/länder (subtitle.contains(",") || subtitle.isEmpty)
    - `resolveLocation(_ completion: MKLocalSearchCompletion) async -> CLPlacemark?` — MKLocalSearch → mapItems.first?.placemark
    - `requestCurrentLocation() async -> CLPlacemark?` — CLLocationManager.requestWhenInUseAuthorization, hämta en position, reverse geocode till CLPlacemark. Använd `CLLocationUpdate.liveUpdates()` (iOS 17+ async API). Returnera nil om användaren nekar tillstånd.

    **OnboardingViewModel.swift** — @Observable @MainActor class:
    - Egenskaper: `displayName: String`, `selectedPhotoItem: PhotosPickerItem?`, `profileImage: UIImage?`, `selectedCity: String`, `selectedCityLatitude: Double?`, `selectedCityLongitude: Double?`
    - Egenskap: `currentStep: Int = 0` (0=namn, 1=foto, 2=plats)
    - Egenskap: `isLoading: Bool = false`
    - `canProceedFromName: Bool` — displayName.trimmingCharacters inte tom
    - `canProceedFromLocation: Bool` — selectedCity inte tom
    - `loadPhoto() async` — ladda bild från PhotosPickerItem
    - `completeOnboarding(uid: String, authManager: AuthManager, userService: UserService) async throws`:
      1. Sätt isLoading = true
      2. Om profileImage finns: upload via userService.uploadProfileImage → få URL
      3. Skapa AppUser med alla fält (displayName, photoURL, city, cityLatitude, cityLongitude, authProvider från Firebase Auth)
      4. Anropa userService.createUserProfile
      5. Uppdatera authManager.currentUser med den nya profilen
      6. Sätt authManager.authState = .authenticated
      7. isLoading = false

    - "Fortsätt"-knapp på varje steg (disabled om canProceed är false)
