<code_context>
## Existing Code Insights

### Reusable Assets
- **NotificationCenter + .openChat pattern**: AppDelegate postar notification, AppRouter lyssnar och sätter binding, MainTabView propagerar till ConversationListView. Samma mönster kan replikeras för weatherAlert.
- **WeatherDetailSheet**: Befintlig vy som tar `FriendWeather` — klar att använda som destination.
- **MainTabView**: Två tabs (Vänner, Chattar) med `@Binding var openConversationId` — liknande binding kan läggas till för weatherAlert.

### Established Patterns
- Push-hantering: `userNotificationCenter(didReceive:)` → checkar `userInfo` → postar NotificationCenter
- Navigation bindings: AppRouter → MainTabView → target view via `@Binding`
- Firestore-skrivning av fcmToken: AppDelegate `messaging(didReceiveRegistrationToken:)` skriver direkt till `users/{uid}`

### Integration Points
- **AppDelegate**: Lägg till weatherAlert-hantering i `userNotificationCenter(didReceive:)`
- **AppRouter**: Ny notification listener (.openWeatherAlert eller liknande)
- **MainTabView**: Ny binding för att öppna väderdetalj på Vänner-tab
- **FriendListView**: Behöver ta emot signal för att visa WeatherDetailSheet för specifik vän
- **AppUser.swift**: Lägg till `var fcmToken: String?` property
