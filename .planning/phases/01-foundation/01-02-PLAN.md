  <name>Task 1: Implementera tre login-metoder i AuthManager</name>
  <files>
    HotAndColdFriends/Core/Auth/AuthManager.swift
  </files>
  <action>
    Implementera de tre login-metoderna i AuthManager (ersätt stubbarna från Plan 01):

    **signInWithApple():**
    - Generera kryptografisk nonce med `SecRandomCopyBytes` och SHA256-hash via CryptoKit
    - Spara raw nonce i en private property `currentNonce`
    - Skapa `ASAuthorizationAppleIDProvider().createRequest()` med requestedScopes [.fullName, .email]
    - Sätt `request.nonce = sha256(nonce)` (hashad version till Apple)
    - Presentera ASAuthorizationController
    - I delegate callback: skapa `OAuthProvider.appleCredential(withIDToken:rawNonce:fullName:)` med RAW nonce (ej hash)
    - Anropa `Auth.auth().signIn(with: credential)`
    - KRITISKT: Apple skickar fullName BARA vid FÖRSTA inloggning — spara displayName i Firestore omedelbart i authorizationController callback. Kolla om displayName redan finns i Firestore (= returning user).

    **signInWithGoogle():**
    - Hämta clientID från `FirebaseApp.app()?.options.clientID`
    - Konfigurera `GIDConfiguration(clientID:)`
    - Hämta rootViewController från UIApplication connectedScenes
    - Anropa `GIDSignIn.sharedInstance.signIn(withPresenting:)` med async/await
    - Skapa `GoogleAuthProvider.credential(withIDToken:accessToken:)`
    - Anropa `Auth.auth().signIn(with: credential)`

    **signInWithFacebook():**
    - Skapa `LoginManager()`
    - Hämta rootViewController
    - Anropa `loginManager.logIn(permissions: ["email", "public_profile"], from:)` wrappat i `withCheckedThrowingContinuation`
    - Kolla att `!result.isCancelled`
    - Skapa `FacebookAuthProvider.credential(withAccessToken: result.token?.tokenString)`
    - Anropa `Auth.auth().signIn(with: credential)`

    Lägg till helper-funktioner:
    - `private func randomNonceString(length: Int = 32) -> String` — använd SecRandomCopyBytes
    - `private func sha256(_ input: String) -> String` — CryptoKit SHA256

    AuthManager behöver conformera till `ASAuthorizationControllerDelegate` och `ASAuthorizationControllerPresentationContextProviding` för Apple Sign-In. Implementera detta som extension.

    Uppdatera `fetchOrCreateUserProfile` för att hantera Apple-namn: om Firestore-profil saknar displayName och Firebase Auth user har displayName → spara den.

    Se till att authState sätts till `.authenticating` innan varje login-anrop och tillbaka till `.unauthenticated` vid fel.
