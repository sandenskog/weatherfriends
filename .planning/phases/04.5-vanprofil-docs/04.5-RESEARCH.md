## Summary

Fasen har två oberoende delar: (1) navigationsimplementation och (2) dokumentationsfix. De kan planeras och exekveras som separata tasks men kan läggas i samma plan.

**Navigationsdelen** handlar om att göra WeatherDetailSheet-headern tappbar och visa vänprofilen som en stackad sheet ovanpå. Befintlig `ProfileView` är nära men fel — den laddar `AppUser` från Firestore via UID, vilket kräver att vännen har ett appkonto. Beslutet är att visa `Friend`-modelldata direkt (displayName, photoURL, city). Det enklaste mönstret är en ny `FriendProfileView` som tar en `Friend` som parameter och visar data direkt utan asynkront Firestore-anrop. Alternativt kan `ProfileView` utökas med ett `Friend`-override — men en separat vy är renare och undviker komplikationer med `ProfileViewModel`.

**Dokumentationsdelen** är en ren textändring: lägg till `requirements-completed: [AUTH-01, AUTH-02, AUTH-03]` i frontmattern på `01-02-SUMMARY.md` och uppdatera PROF-03-raden i `REQUIREMENTS.md` traceability-tabell från "Pending" till "Complete".

**Primary recommendation:** Skapa en ny `FriendProfileView(friend: Friend)` som inte gör nätverksanrop — allt data finns redan i `Friend`-modellen. Gör WeatherDetailSheet-headern tappbar med `.onTapGesture` och presentera vyn som `.sheet(isPresented:)`.

### Pattern 1: FriendProfileView tar Friend direkt

**Vad:** En enkel presentationsvy utan viewmodel eller asynkrona anrop. All data finns redan i `Friend`-modellen som skickas in via WeatherDetailSheet.

**När:** Alltid för vänprofiler — Friend-modellen har displayName, photoURL, city.

**Exempel:**
```swift
struct FriendProfileView: View {
    let friend: Friend

    var body: some View {
        VStack(spacing: 20) {
            profileImageView
                .padding(.top, 24)

            VStack(spacing: 6) {
                Text(friend.displayName)
                    .font(.title2)
                    .fontWeight(.semibold)

                if !friend.city.isEmpty {
                    Label(friend.city, systemImage: "location.fill")
                        .font(.subheadline)
                        .foregroundStyle(.secondary)
                }
            }

            Spacer(minLength: 32)
        }
        .presentationDetents([.medium])
        .presentationDragIndicator(.visible)
    }
}
```

### Pattern 2: Tappbar header i WeatherDetailSheet

**Vad:** Wrappa `headerSection`-innehållet (profilbild + namn + stad) i en `Button` eller lägg till `.onTapGesture`. Lägg till `@State private var showFriendProfile = false` och `.sheet(isPresented:)`.

**Observera:** Enbart profilbild + namn + stad ska vara tappbart (de tre övre elementen). Temperatur och konditiontext är inte del av tap-target enligt beslutet.

### Pattern 3: initialsCircle — kod-duplicering

**Vad:** Både `ProfileView` och `WeatherDetailSheet` har en privat `initialsCircle()`-funktion. `FriendProfileView` behöver samma funktion.

**Rekommendation (Claude's discretion):** Kopiera funktionen in i `FriendProfileView` — det är det enklaste. En gemensam ViewModifier eller extension kan extraheras senare om det känns relevant, men den ökade komplexiteten är inte motiverad för tre platser.

### Pattern 4: Stäng-knapp vs dra-ner

**Vad:** Appen använder konsekvent `.presentationDragIndicator(.visible)` utan stäng-knapp på sheets (WeatherDetailSheet, ProfileView, AddFriendSheet).

**Rekommendation (Claude's discretion):** Ingen stäng-knapp — matcha appens mönster med bara drag-indikator. FriendProfileView presenteras med `.presentationDetents([.medium])` och `.presentationDragIndicator(.visible)` precis som ProfileView.

### Pattern 5: Visuell indikering att headern är tappbar

**Rekommendation (Claude's discretion):** En liten chevron (SF Symbol `chevron.right`) med `.foregroundStyle(.tertiary)` i övre högra hörnet av profilbilden, eller placerad vid sidan av stad-texten. Alternativt inget — headern är redan ett naturligt tap-target. Välj den enklaste lösningen som är konsistent med appens minimalistiska stil: ingen chevron, enbart `buttonStyle(.plain)` utan extra markering. Om UX-feedback behövs kan en `.opacity`-animation läggas till.

### Anti-Patterns to Avoid

- **Ladda AppUser via Firestore för vänprofil:** ProfileViewModel.loadProfile(uid:) fungerar bara för vänner med appkonto (authUid != nil). Friend-data räcker och är redan tillgänglig.
- **Navigationspush istf sheet:** Appen använder sheets genomgående, inte NavigationLink-push för profiler. Håll mönstret.
- **Ändra FriendListView:** Beslutet är att FriendListView-beteendet inte ändras — tap → WeatherDetailSheet är oförändrat.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Profilbildsladdning | Egen cache-logik | AsyncImage (SwiftUI inbyggt) | Redan i bruk i hela appen |
| Sheet-presentation | Programmatisk navigation | .sheet(isPresented:) med @State Bool | Appens etablerade mönster |

## Common Pitfalls

### Pitfall 1: .sheet stacking kräver att parent-vyn äger @State

**Vad som går fel:** Om `showFriendProfile` läggs i fel vy (t.ex. i en sub-view-component), stängs sheet:en direkt när parent:en uppdateras.

**Varför det händer:** SwiftUI sheets måste ägas av den vy som presenterar dem.

**Hur man undviker:** Deklarera `@State private var showFriendProfile = false` direkt i `WeatherDetailSheet`-structen och bifoga `.sheet(isPresented: $showFriendProfile)` på root-nivå i `body` (bredvid `.presentationDetents`).

**Varningssignal:** Sheet visas och försvinner direkt, eller sheet öppnas inte alls.

### Pitfall 2: buttonStyle på tappbar header påverkar textfärg

**Vad som går fel:** En `Button` i SwiftUI ger som standard blå tint-färg på text och ikoner.

**Varför det händer:** Standard `.buttonStyle(.automatic)` applicerar tint-färg.

**Hur man undviker:** Använd `.buttonStyle(.plain)` för att bevara befintlig styling på text och ikoner i headern.

### Pitfall 3: Dokumentationsfix kräver korrekt YAML-frontmatter-format

**Vad som går fel:** YAML-syntax i frontmatter är känslig — felaktigt indrag eller saknat kolon bryter parsning.

**Hur man undviker:** Kopiera exakt format från `01-01-SUMMARY.md` eller `01-03-SUMMARY.md` som har `requirements-completed`-fältet:
```yaml
requirements-completed:
  - AUTH-01
  - AUTH-02
  - AUTH-03
```

### Pitfall 4: Stackade sheets kräver iOS 16.4+ för korrekt funktion

**Vad som går fel:** Stackade sheets (sheet inuti sheet) hade instabilt beteende på iOS 16.x.

**Hur man undviker:** Inga åtgärder behövs — projektet kör iOS 17+ som deployment target. Stackade sheets fungerar stabilt på iOS 17+.

## Code Examples

### FriendProfileView — komplett implementering

```swift
// Source: Projektmönster — matchar ProfileView.swift-strukturen
struct FriendProfileView: View {
    let friend: Friend

    var body: some View {
        VStack(spacing: 0) {
            Spacer().frame(height: 8)
            ScrollView {
                VStack(spacing: 20) {
                    profileImageView
                        .padding(.top, 24)

                    VStack(spacing: 6) {
                        Text(friend.displayName)
                            .font(.title2)
                            .fontWeight(.semibold)

                        if !friend.city.isEmpty {
                            Label(friend.city, systemImage: "location.fill")
                                .font(.subheadline)
                                .foregroundStyle(.secondary)
                        }
                    }

                    Spacer(minLength: 32)
                }
            }
        }
        .presentationDetents([.medium])
        .presentationDragIndicator(.visible)
    }

    @ViewBuilder
    private var profileImageView: some View {
        if let urlString = friend.photoURL, let url = URL(string: urlString) {
            AsyncImage(url: url) { phase in
                switch phase {
                case .success(let image):
                    image.resizable().scaledToFill()
                        .frame(width: 100, height: 100)
                        .clipShape(Circle())
                default:
                    initialsCircle(name: friend.displayName, size: 100)
                }
            }
        } else {
            initialsCircle(name: friend.displayName, size: 100)
        }
    }

    private func initialsCircle(name: String, size: CGFloat) -> some View {
        ZStack {
            Circle()
                .fill(Color(.systemGray5))
                .frame(width: size, height: size)
            Text(initials(from: name))
                .font(.system(size: size * 0.38, weight: .medium))
                .foregroundStyle(.secondary)
        }
    }

    private func initials(from name: String) -> String {
        let parts = name
            .trimmingCharacters(in: .whitespacesAndNewlines)
            .components(separatedBy: .whitespaces)
            .filter { !$0.isEmpty }
        let first = parts.first?.first.map(String.init) ?? ""
        let last = parts.count > 1 ? (parts.last?.first.map(String.init) ?? "") : ""
        let result = (first + last).uppercased()
        return result.isEmpty ? "?" : result
    }
}
```
