  <files>HotAndColdFriends/Features/WeatherCard/WeatherCardBackground.swift</files>
  <action>
Skapa `WeatherCardBackground.swift` med:

1. `enum WeatherCardCategory: String, CaseIterable` med 7 case:
   - `clearDay` (sun-clear)
   - `clearNight` (moon-clear, cloud-moon)
   - `partlyCloudy` (cloud-sun)
   - `overcast` (cloud-overcast, fog, wind)
   - `rain` (rain, heavy-rain, drizzle)
   - `snow` (snow, sleet, hail)
   - `thunderstorm` (thunderstorm)

2. `static func category(for symbolName: String) -> WeatherCardCategory` -- mappar WeatherIconMapper asset-namn till kategori. Anvaend `WeatherIconMapper.assetName(for:)` foerst, sedan matcha. Default: `.overcast`.

3. `@ViewBuilder static func background(for category: WeatherCardCategory) -> some View` -- returnerar en gradient-bakgrund per kategori. Designas med varma, sociala faerger:
   - clearDay: varm gul-till-ljusbla gradient
   - clearNight: morkbla-till-lila gradient med subtila "stjaernor" (sma vita cirklar)
   - partlyCloudy: ljusbla med vit gradient fran oeverdelen
   - overcast: ljusgra till mellangraa gradient
   - rain: blaagra moerk gradient
   - snow: vit till ljusbla isig gradient
   - thunderstorm: moerklila till morkgra gradient

Gradienterna aer placeholder som kan bytas mot riktiga illustrationer senare -- designa dem som `Image("weather-bg-\(category.rawValue)")` med `.resizable()` wrappade i en fallback: foersoek ladda asset, om det saknas visa gradient. Paa saa saett kan Richard laegga till assets naer de aer klara utan kodaendringar.
